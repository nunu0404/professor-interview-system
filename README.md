# 🎓 학과 소개의 날 — 교수 면담 배정 시스템 (Professor Interview Scheduling System)

> 대학교 학과 소개의 날 행사에서 학생들이 희망 연구실(교수)을 선택하고, 학과 측에서 실시간으로 세션 단위로 배정 및 관리할 수 있도록 지원하는 웹 기반 플랫폼입니다.

---

## 📌 주요 핵심 특징

1. **학생 모바일 신청 (QR 기반)**: 행사장 빔프로젝터에 띄워진 대형 QR 코드를 스캔하여 바로 스마트폰 폼으로 접속, 연락처 기반으로 1~3지망 연구실을 빠르고 쉽게 신청합니다.
2. **원클릭 스마트 자동 배정**: 수작업 시 1시간 이상 걸리던 100여명 학생의 지망 별 3개 세션 분산 배정을 버튼 클릭 한 번(약 1초)에 완벽하게 알고리즘으로 자동화합니다.
3. **결함 탐지 및 쉬운 수동 조율**: 시스템 오류나 알고리즘 한계로 공강(구멍)이 생겼거나 지망 외 연구실로 밀려난 불운한 케이스를 대시보드 상단 경고창에서 즉각 감지합니다. 드롭다운 방식으로 손쉽게 핀포인트 수동 조율이 가능합니다.
4. **결과 비공개 보호**: 조율이 완료되기 전까지 학생들은 "조율 중" 대기 화면만 보게 되며, 관리자가 "결과 공개" 초록색 스위치를 켜야만 일제히 결과를 확인할 수 있습니다.
5. **A4 출석부 인쇄 & 엑셀 CSV 추출 지원**: 행사 진행에 필요한 오프라인 명단 인쇄 및 영구 보관용 엑셀 다운로드를 지원합니다.
6. **안정적인 상태 관리**: Next.js App Router의 레이아웃 상태 캐싱으로 인한 화면 백지화 및 무한 로딩 (Hydration) 버그를 방지하여, 관리자 로그인 후 부드러운 화면 전환을 보장합니다.

## 🚀 1. 시스템 설치 및 실행 방법 (초보자 설치 가이드)

### 단계 1: 사전 필수 프로그램 (Node.js) 설치
본 시스템을 실행하려면 컴퓨터에 **Node.js**라는 프로그램이 깔려 있어야 합니다.
- [Node.js 공식 홈페이지](https://nodejs.org/ko/)에 접속하여 **LTS 버전(안정적)**을 다운로드하고, 화면의 기본 설정(Next)대로 쭉 눌러 설치해 주세요.

### 단계 2: 프로그램 전체 코드 다운로드
1. 현재 이 화면(GitHub)의 우측 상단에 있는 초록색 **[<> Code]** 버튼을 클릭합니다.
2. **Download ZIP**을 클릭하여 압축 파일을 다운로드하고, 바탕화면 등 편한 곳에 압축을 풀어줍니다.

### 단계 3: 프로그램 실행하기
윈도우는 **시작 메뉴에서 '명령 프롬프트(CMD)'**를 검색하여 열고, 맥은 **'터미널(Terminal)'** 앱을 엽니다.

1. 압축을 푼 폴더(frontend) 안으로 이동합니다.
   ```bash
   # 예시 (Windows)
   cd Desktop\professor-interview-system-main\frontend

   # 예시 (Mac)
   cd Desktop/professor-interview-system-main/frontend
   ```

2. 필수 패키지를 설치합니다. (최초 1회만 실행하면 됩니다)
   ```bash
   npm install
   ```

3. 시스템 서버를 조용히 가동합니다.
   ```bash
   npm run dev
   ```
   *주의: 명령어를 입력하고 화면에 `Ready in ... ms` 문구가 뜰 때까지 잠시 기다려 주세요. 터미널 창(까만 창)을 끄면 서버가 종료되므로, 행사가 모두 끝날 때 끄시면 됩니다!*

## 💻 2. 관리자 페이지 접속 및 사용법

서버가 가동되었다면, 크롬(Chrome), 엣지(Edge) 등의 인터넷 브라우저에 다음 주소를 입력합니다.

- **관리자 대시보드 주소**: [http://localhost:3000/admin](http://localhost:3000/admin)
- **초기 관리자 비밀번호**: `openlab2026`
- **교수님/연구실 랩장 전용 열람 비밀번호**: `viewer2026` (배정 명단 한정 열람 가능, 조작 권한 없음)

> 운영 중, 행사장에서 "결과 QR 송출" 화면이나 "신청 화면 QR" 화면을 노트북 메인 빔프로젝터에 띄워두시면 본 시스템이 컴퓨터의 로컬 네트워크 IP를 자동으로 추출하여 완벽하게 학생 스마트폰으로 연결해 줍니다. 동일한 와이파이/랜선 네트워크에 접속된 스마트폰이라면 모두 접속 가능합니다.

## 🗓️ 3. 행사 당일 운영 시나리오 (Action Plan)

안전한 행사 시작부터 완벽한 배정 완료까지의 순서입니다. 행정팀 및 조교진은 이 순서를 따라 진행하시면 됩니다.

### [사전 준비] (행사 1~2일 전)
1. 관리자 페이지에 로그인 후 상단의 **[🔬 연구실 관리]** 탭으로 이동합니다.
2. 이번 학과 소개의 날 행사에 참여하는 **모든 연구실(교수님 명, 수용 정원 등)을 등록 및 수정**합니다. (수용 정원은 세션 배정 알고리즘의 핵심 기준이 됩니다.)

### [행사 직전 세팅] (행사 30분 전)
1. 상단 **[⚙️ 설정]** 탭에 들어가서 이전의 연습/모의 데이터가 남아있다면 가장 하단의 **[🗑️ 전체 지우기]** 버튼을 눌러 학생 신청 및 배정 데이터를 새하얗게 초기화합니다. *(※ 등록해 둔 연구실 목록은 지워지지 않으니 안심하셔도 됩니다)*
2. '면담 신청 폼 자동 마감 설정'을 행사 스케줄에 맞게 세팅합니다 (예: 오후 2시면 `14:00`).
3. 행사장의 빔프로젝터나 큰 스크린에 **http://localhost:3000/** 메인 화면을 띄워 대형 QR 코드를 스크린에 노출시킵니다.

### [신청 진행] (모여있는 학생들 대상)
1. 학생들은 빔프로젝터의 QR 코드를 스마트폰 카메라로 찍어 1~3지망을 신청합니다.
2. 관리자는 `/admin` (대시보드) 화면에서 학생들이 어느 연구실에 몰리는지, 신청은 얼마나 들어오고 있는지 경쟁률을 실시간으로 살펴봅니다.

### [신청 마감 및 스피드 배정]
1. 세팅된 시간이 되면 폼이 닫힙니다 (또는 수동으로 우측 상단 [신청 마감] 버튼을 누를 수 있습니다).
2. 관리자 화면 왼쪽 `/admin/assignments` 탭으로 이동하여 파란색 **[⚡ 자동 배정]** 버튼을 과감히 누릅니다. 1초 만에 전체 인원 배정이 끝납니다.
3. 배정 결과 리스트를 눈으로 훑어보고, 화면 위쪽에 **빨간색 경고창(공강 발생, 지망 외 인원)**이 없는지 확인합니다.
4. 배정이 맘에 들지 않거나 누락 케이스가 있다면, 우측 하단의 **연구실 선택 드롭다운**을 눌러 구멍이 난 학생의 빈자리를 적절히 수동으로 변경(조율)해 줍니다.

### [결과 모두에게 공개]
1. 완벽하게 배정을 마쳤다면 `/admin/settings`로 돌아가 **[🔓 결과 공개하기]** 초록색 토글을 클릭해 켭니다.
2. 강당 빔프로젝터 화면을 상단 우측의 **[📱 결과 QR 송출]** 화면으로 변경하여 띄웁니다.
3. 학생들이 해당 스크린의 QR 코드를 찍고 본인의 전화번호와 이름을 치면 각각의 본인 맞춤형 1, 2, 3 세션 시간표(어느 연구실, 어느 교수님 배정인지)를 확인할 수 있습니다.

### [사후 엑셀 백업 및 인쇄]
- 상단 **[🖨️ 배정 명단]**을 누르고 키보드 `Ctrl+P` (맥은 `cmd+P`)를 눌러 각각의 연구실별 명단(출석부)을 예쁘게 A4 용지로 출력합니다. 각 연구실 인솔을 돕는 조교들에게 나눠줍니다.
- 대시보드 우측 상단 **⬇️ 학생 CSV** 및 **⬇️ 배정 CSV** 버튼을 클릭해 클릭 한 번으로 모든 데이터를 엑셀로 다운로드하여 영구 보관/보고용으로 사용합니다.
