# 요구사항 정의서 (Requirements Document)

_작성일: 2026-02-23 | 상태: 초안 (요구사항 수집 전)_

---

## 1. 프로젝트 배경

학과 소개의 날 행사에서 학생들이 교수와 1:1 혹은 소그룹 면담을 희망 우선순위(1지망, 2지망, 3지망)로 신청하고,
학과 행정팀이 이를 효율적으로 배정·관리·운영할 수 있는 시스템이 필요합니다.

---

## 2. 이해관계자

| 역할 | 설명 |
|------|------|
| 학생 | 면담 신청자. 1~3지망 교수 선택 |
| 교수 | 면담 대상자. 일정 및 정원 보유 |
| 학과 관리자 | 신청 수합, 배정, 운영 담당 |

---

## 3. 핵심 기능 (예상 — 추후 확정 필요)

### 3.1 학생용 신청 화면
- [ ] 학생 정보 입력 (이름, 학번 등)
- [ ] 교수 목록 조회 (교수 소개 포함)
- [ ] 1지망 / 2지망 / 3지망 선택
- [ ] 신청서 제출 및 확인

### 3.2 관리자 대시보드
- [ ] 신청 현황 실시간 조회
- [ ] 배정 알고리즘 실행 (우선순위 기반)
- [ ] 배정 결과 수동 조정
- [ ] 결과 학생 통보
- [ ] 데이터 엑셀 내보내기

### 3.3 운영
- [ ] 면담 일정 시간표 관리
- [ ] 교수별 정원 설정
- [ ] 신청 기간 설정 (오픈/마감)

---

## 4. 미확정 사항 (학과 측 확인 필요)

- 신청 인원 규모 (학생 수, 교수 수)
- 면담 형태 (1:1 / 소그룹)
- 시간 슬롯 구성 방식
- 학생 인증 방법 (학번 입력? 로그인? 링크 공유?)
- 배정 기준 (지망 우선 / 선착순 / 랜덤?)
- 결과 공지 방법 (화면 내 확인 / 이메일 / SMS?)
- 데이터 저장 기간 및 보안 요건
